## Getting Started

1. **Set up your development environment**
   - Fork the [Ignix UI repository](https://github.com/mindfiredigital/ignix-ui)
   - Clone your forked repository

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Start the development server**
   ```bash
   npm run dev
   ```

## Selecting an Issue or Feature

1. Check our [issue tracker](https://github.com/mindfiredigital/ignix-ui/issues) for open issues
2. Look for issues you are interested in
3. If you want to work on a new feature, please open an issue first to discuss it with the maintainers

## Making Changes

1. **Create a new branch**
   ```bash
   git checkout -b fix/description-of-fix
   # or
   git checkout -b feature/description-of-feature
   ```

2. **Make your changes**
   - Follow the project's coding style and conventions
   - Write clear commit messages
   - Keep your changes focused and atomic

3. **Add tests**
   - Ensure all tests pass before submitting a PR
   ```bash
   npm test
   ```

## Submitting a Pull Request

1. Push your changes to your fork
   ```bash
   git push origin your-branch-name
   ```

2. Open a Pull Request (PR) against the `development` branch
   - Provide a clear title and description
   - Reference any related issues using `#issue-number`
   - Include screenshots or videos for UI changes

3. Address review feedback
   - Make requested changes
   - Push updates to your branch
   - The PR will be merged once approved
